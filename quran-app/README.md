# دستیار آموزشی قرآن متوسطه اول

PWA آموزش قرآن برای دانش‌آموزان پایه هفتم، هشتم و نهم — ساخته‌شده با React + Vite + TailwindCSS + Supabase.

توسعه‌دهنده: محمد مهدی خراسانی

## نصب و اجرا

```bash
npm install
cp .env.example .env   # سپس مقادیر Supabase را وارد کنید
npm run dev
npm test                # اجرای تست‌های واحد (Vitest)
```

برای راهنمای کامل دیپلوی (Supabase، Edge Functions، Vercel/Netlify) به `DEPLOYMENT.md` مراجعه کنید.

## ساختار پروژه

```
src/
  components/
    layout/       # Header، BottomNav، Layout اصلی
    lessons/       # LessonCard و کامپوننت‌های مرتبط با درس
  pages/           # StudentDashboard، LessonsList، LessonDetail، Profile
  lib/
    supabaseClient.js   # اتصال به Supabase
    mockData.js          # داده‌های نمونه + منطق قفل توالی دروس
supabase/
  schema.sql       # ساختار کامل پایگاه داده + RLS
```

  `bulk-import-students`، از یک فایل CSV (نام، **کد ملی**، کد دانش‌آموزی، شمارهٔ تماس والدین)
  برای هر ردیف یک حساب واقعی Supabase Auth می‌سازد. **نام کاربری = کد ملی**، **رمز عبور پیش‌فرض
  = کد دانش‌آموزی**. صفحهٔ ورود اکنون دو تب دارد: «ورود دانش‌آموز» (با کد ملی) و «ورود معلم/مدیر»
  (با ایمیل). این قابلیت فقط در پنل ادمین (`/admin/users`، تب دانش‌آموزان) در دسترس است و برای
  هر کلاس جداگانه اجرا می‌شود (مثلاً ۱۲ بار برای ۱۲ کلاس)؛ **معلم فقط نقش نظارتی** دارد و در
  `/teacher/students` فقط می‌تواند پیشرفت دانش‌آموزان را مشاهده کند، نه افزودن/حذف. پیش از
  استفاده، مایگریشن `0003_add_national_id.sql` باید اجرا شده باشد.
- ✅ تست E2E اولیه با Playwright (`tests/e2e/critical-flows.spec.js`) برای مسیرهای ورود و دسترسی نقش‌ها
- ✅ **مدیریت واقعی محتوای دروس**: پنل ادمین (`/admin/lessons`) اکنون واقعاً به جدول
  `lessons` در Supabase وصل است — متن قرآن، ترجمه، هر ۳ فعالیت، انس روزانه، و **فایل صوتی

  ذخیره می‌شوند. صفحهٔ درسِ دانش‌آموز (`LessonDetail.jsx`) این محتوا را در بارگذاری صفحه
  واکشی می‌کند و در نبود محتوای واقعی، به‌طور شفاف به داده‌های نمونهٔ mockData برمی‌گردد.
  پیش‌نیاز: مایگریشن `0004_lesson_audio.sql` (ستون `audio_url` + باکت `lesson-audio`).
- ✅ **رفع ناهماهنگی شناسهٔ درس**: `src/lib/lessonsApi.js` اکنون UUID واقعی هر درس را هم برمی‌گرداند
  (`dbId`). صفحهٔ آزمون و ضبط تلاوت، پیش از فراخوانی Edge Functionهای واقعی (`generate-quiz`،
  `process-recitation`)، این UUID واقعی را واکشی می‌کنند؛ اگر درس هنوز در دیتابیس ثبت نشده باشد
  (فقط در mockData وجود دارد)، به‌طور خودکار به شبیه‌سازی محلی برمی‌گردند، بدون خطای نوع داده.
- ✅ **مقاوم‌سازی در برابر ترافیک هم‌زمان**: لایهٔ `api.js` اکنون تا ۲ بار با مکث فزاینده (retry با
  backoff) تلاش می‌کند، برای مواقعی که چند دانش‌آموز هم‌زمان درخواست AI می‌فرستند و ممکن است
  به‌طور موقت به محدودیت نرخ OpenAI بخورند.
- ✅ **پیشرفت واقعی دانش‌آموز**: هر سه فعالیت، نمرهٔ تلاوت و نمرهٔ آزمون اکنون واقعاً در جدول
  `student_progress` ذخیره می‌شوند (`src/lib/progressApi.js` + upsert در Edge Functionهای
  `process-recitation`/`grade-quiz`) و با بازگشت دانش‌آموز به همان درس (حتی از دستگاه دیگر)
  بازیابی می‌شوند. این فقط برای دانش‌آموزان واقعاً واردشده (نه حالت دمو) و دروسی که واقعاً در
  دیتابیس ثبت شده‌اند فعال است؛ در غیر این صورت به‌طور خاموش فقط در state محلی می‌ماند.
- ✅ **مدیریت واقعی کاربران در پنل ادمین**: Edge Function ششم، `admin-manage-users`، فهرست واقعی
  کاربران هر نقش را از Supabase می‌خواند و افزودن/حذف واقعی (نه فقط محلی) انجام می‌دهد؛ حذف کاربر
  به‌خاطر `ON DELETE CASCADE` در اسکیما، خودکار پروفایل/دانش‌آموز/معلم مرتبط را هم پاک می‌کند.
